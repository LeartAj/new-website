2. LOADING SCREEN FEATURE
==============================================================================

2.1 Loading Screen Display
- Full-screen overlay that covers entire viewport
- Contains an animated 3D rotating sphere made of meridians, latitudes, and axes
- Sphere consists of:
  * 36 meridian circles (rotated at 10-degree increments from 10° to 360°)
  * 12 latitude circles (positioned at different z-depths)
  * 2 axis lines

2.2 Loading Screen Animation
- Sphere rotates continuously using CSS keyframe animation
- 8-second rotation cycle
- 3D transforms: rotateX(30deg), rotateY(0-360deg), rotateZ(80deg)

2.3 Session-Based Display Logic
CRITICAL FUNCTIONALITY:
- Loading screen shows ONLY ONCE per browser session
- Uses sessionStorage with key 'loadingShown'
- Inline script immediately after loading screen div checks sessionStorage
- If 'loadingShown' exists, set display:none immediately (prevents flash)
- After 1 second delay on first load, add 'hidden' class for fade-out
- Set sessionStorage 'loadingShown' = 'true' after first display

Script flow:
1. Inline script right after loading-screen div: check sessionStorage, hide if shown
2. On window load event: if not shown before, wait 1s, fade out, mark as shown

- On window scroll > 50px:
  * Change navbar background to --navbar-scroll variable
  * Add box shadow: '0 2px 20px var(--shadow)'
- On scroll <= 50px:
  * Revert to --navbar-bg
  * Remove box shadow

Active Page Highlighting:
- Get current page from window.location.pathname
- Compare with each nav-link href
- Add 'active' class to matching link

Section Highlighting on Scroll:
- Find all sections with id attributes
- On scroll, check which section is in viewport
- Update nav links for hash-based navigation
- Only update links that start with # (section links)

Scroll Animations:
- Use IntersectionObserver for card animations
- Threshold: 0.1
- Root margin: '0px 0px -50px 0px'
- Add 'visible' class when elements enter viewport

Element-specific:
